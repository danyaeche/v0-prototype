# ALSO — clickable prototype

Pixel-faithful rebuild of the 6 ALSO screens, fully wired for click-through, branded with
the ALSO woven-coil logo and the "Industrial Logistics" design system, and structured to
import into Figma. The part viewer is a **real-time 3D CAD model** (Three.js).

## Screens
| # | File | Screen |
|---|------|--------|
| 01 | `parts.html` | Brand workspace · parts list |
| 02 | `part-detail.html` | Part detail · **live 3D CAD viewer** · comments · quotes |
| 03 | `supplier-portal.html` | Supplier portal · magic-link landing |
| 04 | `quote-submission.html` | Quote submission · revision-linked |
| 05 | `login.html` | Login |
| 06 | `signup.html` | Sign up |

`index.html` redirects to login. `styles.css` is the shared design system. `logo.svg` is the
ALSO mark.

## Design system (wired into `styles.css` as CSS variables)
- **Colors** — Logistics Green `#009571` (accent/CTA), Charcoal `#1c1c1c` (text + dark panels),
  Steel Gray `#7a7a7a`, Border `#d0d3d3`, Concrete `#f4f4f4` (bg), Surface `#fcfcfc`.
- **Type** — Inter for body/UI; **Space Grotesk** for headings as a free stand-in for
  *Laygrotesk* (swap `--font-head` when you license the real face).
- **Shape** — radii sm `4px` · md `8px` · lg `24px`; 4px-based spacing.

Change any `--*` token at the top of `styles.css` to restyle the whole system at once.

## Real-time 3D CAD viewer (screen 02)
`part-detail.html` renders an interactive metallic part with **Three.js** (vendored locally —
see "Runs fully offline" below): PBR aluminium material, environment-map reflections,
orbit + scroll-zoom, and a working toolbar (zoom, fit/reset, auto-rotate toggle, grid toggle).

**It loads a real CAD file.** On open it fetches `models/top-tube-assembly.stl` — an actual
binary STL (1,568 triangles) generated by `models/generate_stl.py`. Re-run that script to
regenerate or tweak the geometry.

**Load your own CAD file**, three ways — all auto-centre and scale to fit:
- Drag a `.stl`/`.glb`/`.gltf`/`.obj` onto the viewer, or use the **Load CAD** button (top-right).
- Click **Upload CAD** on the Parts page (screen 01) — it opens a file picker, then loads your
  file straight into the viewer (handed off via IndexedDB as a Blob in `cadstore.js`, so large
  files work — no practical size cap).

STL/OBJ get the aluminium material; glTF keeps its own materials. STEP/IGES aren't
browser-native — export to STL or glTF first (Fusion 360, SolidWorks, FreeCAD all do this),
or send me the STEP and I'll convert it.

### Runs fully offline
Three.js (core + OrbitControls, RoomEnvironment, the STL/GLTF/OBJ loaders, and
BufferGeometryUtils) is **vendored locally** under `vendor/three/` — no CDN. The importmap in
`part-detail.html` points at those local files. The only remaining network use in the prototype
is Google Fonts; offline, type falls back to system fonts and everything still works.

## Run it (clickable prototype)
```bash
cd "chorus-prototype" && python3 -m http.server 4599
```
Open http://localhost:4599 — best viewed at ≥1280px wide. A floating **Screens** pill
(bottom-right) jumps between all 6 screens; every realistic link is wired.

## Import into Figma (editable layers)
Use the free **html.to.design** plugin (Figma → Plugins → "html.to.design"):
1. **From URL** — run the server above, paste each screen URL (e.g.
   `http://localhost:4599/parts.html`) into the plugin. Import at ~1280px width so tables stay
   single-row. Each screen becomes a native, editable Figma frame.
2. After import, delete the bottom-right "Screens" pill layer (tagged `data-figma-skip`) and
   wire Figma's native prototype arrows between frames.

**3D caveat:** the live 3D canvas can't import as editable vectors — Figma captures only a
still. For the Figma version of screen 02, drop in a rendered image of the part instead
(I can export one on request).
